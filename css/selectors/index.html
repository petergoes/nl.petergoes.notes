CSS3 spec - selectors
MDN - CSS selectors

Basic selectors:

  • *: Matches any element
  • E: Matches a specified element
  • .class: Matches an element with a specified class attribute
  • #id: Matches an element with a specified id attribute
  • E[attribute]: Matches an element with a specified attribute
  • E[attribute=value]: Matches an element with a specific equal attribute / value pair
  • E[attribute~=value]: Matches an element with an attribute that includes whitespace-seperated word equal to the value
  • E[attribute|=value]: Matches an element with an attribute that includes hyphen-seperated list of values starting with the value
  • E[attribute^=value]: Matches an element with an attribute where the value starts with exactly the value
  • E[attribute$=value]: Matches an element with an attribute where the value ends with exactly the value
  • E[attribute*=value]: Matches an element with an attribute that contains the value somewhere.

Combinators:

  • E F: Matches the descendant F of element E
  • E > F: Matches the child F of element E
  • E + F: Matches the adjacent sibling F of element E
  • E ~ F: Matches the general sibling F of element E

Pseudo classes:

  • :active: element that is being activated by the user
  • :checked: a user interface element E which is checked (for instance a radio-button or checkbox)
  • :disabled: a user interface element E which is disabled
  • :empty: an E element that has no children (including text nodes)
  • :enabled: a user interface element E which is enabled
  • :first-child: an E element, first-child of its parent
  • :first-of-type: an E element, first sibling of its type
  • :focus: element that has focus
  • :hover: element that is being hovered over by the user
  • :in-range: element whose current value is within the range limits specified by the min and max attributes
  • :indeterminate: element whose state is indeterminate
  • :invalid: element whose contents fail to validate
  • :lang(nl): an element of type E in language nl
  • :last-child: an E element, last child of its parent
  • :last-of-type: an E element, last sibling of its type
  • :link: element that has not yet been visited
  • :not(): an E element that does not match a simple selector
  • :nth-child(n): an E element, the n-th child of its parent
  • :nth-last-child(n): an E element, the n-th child of its parent, counting from the last one
  • :nth-last-of-type(): an E element, the n-th sibling of its type
  • :nth-of-type(n): an E element, the n-th sibling of its type, counting from the last one
  • :only-child: an E element, only child of its parent
  • :only-of-type: an E element, only sibling of its type
  • :optional: element that does not have the required attribute set on it
  • :out-of-range: element whose current value is outside the range limits specified by the min and max attributes
  • :read-only: element (such as a locked text input) that is not editable by the user
  • :read-write: element (such as a text input) that is editable by the user
  • :required: element that has the required attribute set on it
  • :root: an E element, root of the document
  • :target: an E element being the target of the referring URI
  • :valid: element whose contents validate successfully
  • :visited: element that has been visited

Pseudo elements:

  • ::after: Generates an pseudo element after the elements content
  • ::before: Generates an pseudo element before the elements content
  • ::first-letter: Matches the first formatted letter of an element
  • ::first-line: Matches the first formatted line of an element
  • ::selection: Applies styles to the portion of a document that has been highlighted by the user
